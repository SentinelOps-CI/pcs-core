# PCS v0.1.0 release checklist

The public documentation index is [../README.md](../README.md).

## Objective

Confirm the PCS v0.1 trust loop is complete, reproducible, schema-validated, and protected against fixture drift across LabTrust QC release, tool-use safety, computation reproducibility, and benchmark ingest.

## Artifact chain (LabTrust)

```
LabTrust-Gym runtime trace
  → RuntimeReceipt.v0
  → TraceCertificate.v0
  → ScienceClaimBundle.v0
  → VerificationResult.v0
  → SignedScienceClaimBundle.v0
  → Scientific Memory import report
```

**Canonical fixtures:** `examples/labtrust-release/` — see [../labtrust-release-fixtures.md](../labtrust-release-fixtures.md) and [../release-protocol.md](../release-protocol.md).

## Validate (from repository root)

**One command (Windows):** `powershell -File scripts/run-release-verify.ps1`

**One command (Linux/macOS/Git Bash):** `bash scripts/run-release-verify.sh` or `just release-verify`

**CI parity (includes build + lint):** `just ci`

Manual steps run after `pip install -e python/.[dev]`.

```bash
cd python

pcs schema check
pcs examples check
python -m pcs_core.hash_vectors --verify
pcs shared-hash-vectors verify

pcs validate-release-chain ../examples/labtrust-release/
pcs validate-release-chain ../examples/tool-use-release/
pcs validate-release-chain ../examples/computation-release/
pcs validate ../examples/labtrust-release/release_manifest.v0.json

pcs registry validate ../examples/artifact_registry.valid.json
pcs registry audit

pcs conformance run --suite all
pcs conformance run --suite multidomain
pcs conformance run --suite benchmark-ingest
pcs conformance run --suite benchmark-report

pcs benchmark validate
pcs benchmark validate-ingest --release-grade
python ../scripts/validate_benchmark_ingest_examples.py --release-grade

pytest -q
pytest -q tests/test_release_chain.py tests/test_protocol_conformance.py
```

```bash
cd rust && cargo test && cargo test shared_hash_vectors
cd typescript && npm test
```

Regenerate LabTrust protocol artifacts when pins change.

```bash
just materialize-labtrust-protocol
```

`release_manifest.v0.json` pins `release_chain_validation_result.sha256` to the on-disk validation artifact, and maintainers regenerate both files together through `just materialize-labtrust-protocol`.

## Pin downstream repositories

Tag pcs-core **`v0.1.0`**. Consumer repositories should pin the same pcs-core commit (or tag) and run `pcs validate` / `pcs conformance` in CI.
